# PDF翻译工具需求文档

## 项目概述

### 项目名称
PDF翻译工具

### 项目目标
创建一个支持多翻译API的PDF翻译工具，能够准确提取PDF文本，高质量翻译，并生成格式良好的新PDF。

### 项目背景
用户需要将PDF文档从一种语言翻译成另一种语言，现有的翻译工具大多只支持纯文本翻译，无法保留PDF的原始格式和布局。本工具旨在解决这一问题，提供高质量的PDF翻译服务。

## 功能需求

### 核心功能

#### 1. PDF文本提取
- 支持提取普通文本内容
- 保留文本位置信息
- 支持提取指定页面文本
- 智能识别并过滤页眉、页脚和页码

#### 2. 多翻译API支持
- aiping翻译API
- 硅基流动翻译API
- 支持切换不同翻译服务
- 支持批量翻译

#### 3. 文档生成
- **PDF生成**：
  - 基于原始PDF生成新PDF
  - 保留原始布局和格式
  - 支持绘制翻译后的文本
  - 支持处理表格内容
  - 字体自动适配
- **Word生成**：
  - 基于合并后的翻译结果生成Word文档
  - 保留原始字体、大小、颜色和样式
  - 支持图片提取和插入
  - 支持页面分节和分页符
  - 不包含原PDF中不存在的内容（如页码标题）

#### 4. 指定页码翻译
- 支持翻译指定页码
- 支持翻译页码范围
- 支持混合页码选择
- 提高翻译效率

### 用户界面功能

#### 1. 文件上传
- 支持拖拽上传
- 支持点击上传
- 显示上传进度
- 支持PDF文件预览

#### 2. 参数设置
- 源语言选择
- 目标语言选择
- 翻译服务选择
- 页码范围指定
- 输出格式选择（PDF、Word或两者都输出）
- 术语表配置（可选）

#### 3. 进度显示
- 实时翻译进度
- 当前处理阶段显示
- 取消翻译按钮
- 预计剩余时间

#### 4. 结果处理
- 翻译结果预览
- 文件下载
- 翻译历史记录

## 非功能需求

### 性能需求
- 文本提取：支持100页以上的PDF文档
- 翻译速度：每页翻译时间不超过30秒（取决于翻译API响应速度）
- 响应时间：Web界面响应时间不超过2秒
- 并发处理：支持多用户同时使用

### 可靠性需求
- 错误处理：友好的错误提示和处理机制
- 数据完整性：确保翻译过程中数据不丢失
- 资源清理：及时清理临时文件
- 任务管理：支持翻译任务取消

### 安全性需求
- API密钥安全：使用环境变量管理API密钥
- 文件安全：上传文件安全存储
- 用户隐私：不保存用户上传的原始文件

### 可维护性需求
- 模块化设计：清晰的代码结构
- 日志记录：详细的日志便于调试
- 测试覆盖：单元测试覆盖率不低于80%
- 文档完善：完整的开发文档和使用说明

## 附录

### 术语表
- **TextBlock**：文本块，PDF中的最小文本单位
- **Semantic Block**：语义块，包含相关内容的多个文本块的组合
- **Body Text**：正文文本，需要翻译的内容
- **Non-body Text**：非正文文本，如页眉、页脚、页码等
