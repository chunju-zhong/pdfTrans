## 解决AI Ping LLM调用文本块拆分导致翻译语义断裂问题的优化计划

### 1. 目标
解决AI Ping LLM调用时 "文本块拆分导致翻译语义断裂" 的问题，通过语义块合并、优化提示词和结果拆分映射，确保翻译语义连贯，同时适配后续样式渲染。

### 2. 实现步骤

#### 2.1 语义块合并（app.py）

1. **修改process_translation函数**：在翻译前添加语义块合并逻辑
2. **合并算法**：
   - 遍历所有页面的所有块
   - 按 "位置相邻 + 内容完整" 合并成完整段落
   - 记录原始块与合并块的映射关系
   - 保留原始块的元数据，方便后续拆分

3. **代码实现**：
   ```python
   # 语义块合并逻辑
   def merge_semantic_blocks(all_blocks):
       merged_blocks = []
       block_mapping = []  # 记录原始块与合并块的映射关系
       
       if not all_blocks:
           return merged_blocks, block_mapping
       
       current_merged = {
           'block_text': all_blocks[0]['block_text'],
           'original_blocks': [all_blocks[0]]
       }
       
       for i in range(1, len(all_blocks)):
           prev_block = all_blocks[i-1]
           curr_block = all_blocks[i]
           
           # 位置相邻检查（垂直距离小于阈值）
           prev_bbox = prev_block['original_block']['block_bbox']
           curr_bbox = curr_block['original_block']['block_bbox']
           vertical_distance = curr_bbox[1] - prev_bbox[3]  # y0 - previous y1
           
           # 内容完整检查（当前块不是完整句子结束）
           prev_ends_with_sentence = prev_block['block_text'].strip().endswith(('.', '!', '?', '。', '！', '？'))
           
           if vertical_distance < 10 and not prev_ends_with_sentence:
               # 合并块
               current_merged['block_text'] += ' ' + curr_block['block_text']
               current_merged['original_blocks'].append(curr_block)
           else:
               # 保存当前合并块
               merged_blocks.append(current_merged)
               block_mapping.append(current_merged['original_blocks'])
               # 开始新的合并块
               current_merged = {
                   'block_text': curr_block['block_text'],
                   'original_blocks': [curr_block]
               }
       
       # 添加最后一个合并块
       merged_blocks.append(current_merged)
       block_mapping.append(current_merged['original_blocks'])
       
       return merged_blocks, block_mapping
   ```

#### 2.2 优化提示词（translator.py和各个翻译器实现）

1. **更新系统提示词**：针对技术文档英翻中设计专属提示词
2. **强调关键点**：语义连贯、术语一致、句式严谨
3. **适配Qwen3-32B**：利用其128K上下文优势

4. **代码实现**：
   ```python
   # 在translate_with_context方法中更新提示词
   system_prompt = f"""
   你是专业的{doc_type}翻译专家，擅长将{source_lang_name}文档精准翻译成{target_lang_name}，严格遵循以下规则：
   1. 语义连贯：必须基于完整上下文理解，禁止逐词直译导致的句子断裂；
   2. 术语一致：严格使用以下术语表翻译，不随意变更：
      {glossary if glossary else '无'}
   3. 句式严谨：保持技术文档的正式语气，保留被动语态和逻辑连接词；
   4. 不增删义：严格遵循原文含义，不添加额外解释，不遗漏任何细节（包括标点、括号、冒号的位置）；
   5. 格式兼容：翻译结果需便于后续按原文本块拆分，不擅自添加换行、分段（保持段落级完整性）。
   """
   ```

#### 2.3 结果拆分映射（app.py）

1. **实现拆分算法**：将完整翻译结果按原文本块的长度比例拆分
2. **确保一一对应**：更新每个原始块的翻译内容
3. **保持样式渲染兼容性**：不影响后续样式渲染

4. **代码实现**：
   ```python
   # 结果拆分映射逻辑
   def split_translated_result(merged_translation, original_blocks):
       translated_blocks = []
       
       # 计算原始块总长度和每个块的长度比例
       total_original_length = sum(len(block['block_text']) for block in original_blocks)
       
       if total_original_length == 0:
           return ['' for _ in original_blocks]
       
       # 按长度比例拆分翻译结果
       start_pos = 0
       for i, block in enumerate(original_blocks):
           # 计算当前块应分配的翻译长度
           block_ratio = len(block['block_text']) / total_original_length
           block_trans_len = int(len(merged_translation) * block_ratio)
           
           # 处理最后一个块，分配剩余所有内容
           if i == len(original_blocks) - 1:
               block_trans = merged_translation[start_pos:].strip()
           else:
               block_trans = merged_translation[start_pos:start_pos+block_trans_len].strip()
               # 尝试在句子边界拆分
               for j in range(block_trans_len, min(len(merged_translation), start_pos+block_trans_len+50)):
                   if merged_translation[j] in ('.', '!', '?', '。', '！', '？'):
                       block_trans = merged_translation[start_pos:j+1].strip()
                       start_pos = j+1
                       break
               else:
                   # 未找到合适的句子边界，按比例拆分
                   start_pos += block_trans_len
           
           translated_blocks.append(block_trans)
       
       return translated_blocks
   ```

#### 2.4 更新翻译流程（app.py）

1. **修改process_translation函数**：
   - 添加语义块合并步骤
   - 使用合并后的块进行翻译
   - 将翻译结果拆分回原始块
   - 更新translated_content中的块内容

2. **代码实现**：
   ```python
   # 语义块合并
   merged_blocks, block_mapping = merge_semantic_blocks(all_blocks)
   
   # 翻译合并后的块
   for i, merged_block in enumerate(merged_blocks):
       # 调用翻译API
       merged_translation = translator.translate_with_context(
           {
               'block_text': merged_block['block_text'],
               'block_no': 0,
               'page_num': merged_block['original_blocks'][0]['page_num']
           },
           {'previous_blocks': [], 'next_blocks': []},  # Qwen3-32B支持长上下文，无需额外上下文
           source_lang,
           target_lang,
           doc_type=doc_type,
           glossary=glossary
       )
       
       # 拆分翻译结果
       original_blocks = block_mapping[i]
       translated_block_texts = split_translated_result(merged_translation, original_blocks)
       
       # 更新原始块的翻译内容
       for j, block_text in enumerate(translated_block_texts):
           original_block = original_blocks[j]
           translated_block = original_block['original_block'].copy()
           translated_block['block_text'] = block_text
           
           # 添加到对应页面的翻译结果中
           page_num = original_block['page_num']
           page_translated_blocks_dict[page_num]['blocks'].append(translated_block)
   ```
