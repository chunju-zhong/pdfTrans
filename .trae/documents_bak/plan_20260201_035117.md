# 表格提取和生成优化计划

## 问题分析

在当前实现中，表格处理存在以下问题：

1. **单元格大小信息缺失**：表格提取时只获取整个表格的边界框，没有获取每个单元格的大小信息
2. **表格绘制位置和大小不准确**：表格生成时使用固定的默认值或简单计算，导致表格位置偏上、大小偏小
3. **表格结构信息不完整**：缺少行高、列宽等结构信息，影响表格的精确重绘

## 修复方案

### 1. 扩展数据结构

**修改文件**：`models/extraction.py`

* 创建 `PdfCell` 类，包含单元格的文本、边界框和大小信息

* 修改 `PdfTable` 类，**移除** **`content`** **属性**，添加 `cells` 属性存储每个单元格的信息

* 为 `PdfTable` 类添加 `row_heights` 和 `col_widths` 属性，存储行高和列宽信息

### 2. 修改表格提取逻辑

**修改文件**：`modules/pdf_extractor.py`

* 使用 pdfplumber 的 `table.cells` 属性获取单元格信息

* 为每个单元格计算边界框和大小

* 构建 `PdfCell` 对象列表并存储到 `PdfTable` 的 `cells` 属性中

* 计算行高和列宽信息，存储到 `PdfTable` 的 `row_heights` 和 `col_widths` 属性中

* **移除** **`content`** **相关代码**，直接使用 `cells` 存储表格数据

### 3. 修改表格翻译逻辑

**修改文件**：`services/translation_service.py`

* 翻译表格时直接处理 `cells` 中的单元格信息

* 在 `translated_table` 中添加 `cells` 字段，保存翻译后的单元格信息

* 添加 `row_heights` 和 `col_widths` 字段，保存表格结构信息

* **移除** **`content`** **相关代码**，直接使用 `cells` 存储翻译后的表格数据

### 4. 优化表格生成逻辑

**修改文件**：`modules/pdf_generator.py`

* 修改 `_draw_translated_table` 方法，使用 `cells` 中的边界框信息进行绘制

* 利用 `row_heights` 和 `col_widths` 信息，实现精确的表格结构重绘

* 移除对 `content` 的依赖，完全基于 `cells` 进行表格绘制

* 优化字体处理，确保文字在单元格内正确显示

### 5. 测试和验证

* 创建包含各种表格类型的测试 PDF 文件

* 测试表格提取的准确性，验证单元格大小信息是否正确获取

* 测试表格翻译的完整性，确保单元格信息在翻译过程中不丢失

* 测试表格生成的精确性，验证表格位置和大小是否与原始表格一致

* 测试不同语言的表格文字显示，确保字体处理正确

## 预期效果

1. **精确的表格提取**：能够获取每个单元格的大小和位置信息
2. **完整的表格结构**：保存行高、列宽等结构信息
3. **准确的表格重绘**：表格位置和大小与原始表格一致
4. **稳定的字体处理**：表格文字显示正确，无问号或乱码
5. **简化的数据结构**：移除冗余的 `content` 属性，使用更完整的 `cells` 属性

## 实施步骤

1. **扩展数据结构**：修改 `models/extraction.py` 文件，创建 `PdfCell` 类并修改 `PdfTable` 类
2. **修改提取逻辑**：更新 `modules/pdf_extractor.py` 文件，添加单元格信息提取
3. **修改翻译逻辑**：更新 `services/translation_service.py` 文件，处理翻译后的单元格信息
4. **优化生成逻辑**：更新 `modules/pdf_generator.py` 文件，使用单元格信息进行绘制
5. **测试验证**：运行测试，验证修改效果
6. **文档更新**：更新相关文档，说明新的表格处理功能

## 技术要点

* **pdfplumber 库的高级使用**：利用 `table.cells` 属性获取单元格信息

* **数据结构设计**：合理设计 `PdfCell` 类，提供完整的单元格信息

* **精确的坐标计算**：确保单元格位置和大小的准确计算

* **字体处理优化**：使用适合目标语言的字体，确保文字显示正确

* **代码简化**：移除冗余的 `content` 属性，简化数据结构和处理逻辑
