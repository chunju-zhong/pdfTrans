# 实现上下文感知的翻译机制

## 问题分析
当前翻译流程中，每个文本块独立翻译，导致上下文丢失，一句话可能被分割到多个文本块甚至多个页面中，翻译结果不准确。

## 解决方案
实现上下文感知的翻译机制，给LLM提供当前翻译块的前后几个块作为翻译参考，包括跨页的情况，提高翻译准确性。

## 实现思路

### 1. 修改翻译流程
- 在 `app.py` 的 `process_translation` 函数中，添加上下文收集逻辑
- 为每个翻译块提供前2个和后2个块作为上下文
- **特别处理**：当前页的最后几个块需要包含下一页的开头几个块作为上下文
- **特别处理**：下一页的开头几个块需要包含上一页的最后几个块作为上下文
- 将上下文与当前块一起传递给翻译API

### 2. 翻译器接口扩展
- 为翻译器添加支持上下文的翻译方法
- 修改 `translator.py` 基类，添加 `translate_with_context` 方法
- 在具体翻译器实现中支持上下文参数

### 3. 上下文格式设计
- 上下文包含当前块、前两个块和后两个块
- **跨页处理**：对于页尾块，后两个块可能来自下一页
- **跨页处理**：对于页头块，前两个块可能来自上一页
- 每个块包含块文本、块编号和页码
- 使用清晰的分隔符区分不同块和页码

### 4. 翻译结果处理
- 解析翻译结果，提取当前块的翻译
- 考虑是否需要调整之前已翻译的内容
- 保持文本块之间的连贯性

## 关键修改点

1. **`modules/translator.py`**：
   - 添加 `translate_with_context` 方法到基类
   - 实现默认的上下文处理逻辑

2. **`modules/baidu_translator.py`**：
   - 实现 `translate_with_context` 方法

3. **`modules/aiping_translator.py`**：
   - 实现 `translate_with_context` 方法

4. **`modules/silicon_flow_translator.py`**：
   - 实现 `translate_with_context` 方法

5. **`app.py`**：
   - 修改 `process_translation` 函数，添加上下文收集
   - **添加跨页上下文处理逻辑**
   - 将上下文传递给翻译器
   - 处理带上下文的翻译结果

## 测试计划

### 1. 测试文件位置
- **单元测试**：`tests/test_translator.py`
- **集成测试**：`tests/test_context_translation.py`

### 2. 测试用例设计

#### 2.1 基本上下文翻译测试
- **测试目的**：验证带上下文的翻译方法是否正常工作
- **测试步骤**：
  - 创建翻译器实例
  - 调用 `translate_with_context` 方法
  - 验证翻译结果的准确性
- **预期结果**：翻译结果包含当前块的正确翻译，忽略上下文块

#### 2.2 跨块上下文翻译测试
- **测试目的**：验证跨块上下文能提高翻译准确性
- **测试步骤**：
  - 准备一个被分割到多个块的句子
  - 分别使用普通翻译和上下文翻译
  - 比较两种翻译结果的准确性
- **预期结果**：上下文翻译结果更准确，保持句子连贯性

#### 2.3 跨页上下文翻译测试
- **测试目的**：验证跨页上下文处理是否正确
- **测试步骤**：
  - 准备一个被分割到两个页面的句子
  - 测试页尾块的翻译，确保包含下一页的上下文
  - 测试页头块的翻译，确保包含上一页的上下文
- **预期结果**：跨页句子的翻译保持连贯性和准确性

#### 2.4 边界情况测试
- **测试目的**：验证边界情况的处理
- **测试步骤**：
  - 测试第一页第一个块的翻译
  - 测试最后一页最后一个块的翻译
  - 测试只有一个块的页面
- **预期结果**：所有边界情况都能正确处理，不崩溃

#### 2.5 上下文大小测试
- **测试目的**：验证不同上下文大小的效果
- **测试步骤**：
  - 使用不同大小的上下文（1-3个前后块）
  - 比较翻译结果的准确性
- **预期结果**：适当大小的上下文能提高翻译质量，过大的上下文影响不大
