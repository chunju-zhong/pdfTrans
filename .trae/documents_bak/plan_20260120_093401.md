## 添加日志监控 blocks 信息的生成和传递

### 关键节点

1. **blocks 生成节点**：在 `pdf_extractor.py` 的 `extract_text()` 方法中添加日志，监控 blocks 信息的生成
2. **blocks 传递节点**：在 `app.py` 的 `process_translation` 函数中添加日志，监控 blocks 信息的传递
3. **blocks 使用节点**：在 `pdf_generator.py` 的 `generate_pdf()` 方法中添加日志，监控 blocks 信息的使用

### 具体修改

1. **pdf_extractor.py**
   - 在 `extract_text()` 方法中添加日志，记录生成的 blocks 数量
   - 日志内容：页码、blocks 数量、每个 block 的简要信息

2. **app.py**
   - 在 `process_translation` 函数中添加日志，记录从 `extracted_content` 中获取的 blocks 信息
   - 日志内容：任务 ID、blocks 数量、传递情况
   - 在调用 `pdf_generator.generate_pdf()` 前添加日志，记录传递给生成器的 blocks 信息

3. **pdf_generator.py**
   - 在 `generate_pdf()` 方法中添加日志，记录接收到的 blocks 信息
   - 日志内容：原始文件、输出文件、接收到的 blocks 数量
   - 在块级关联逻辑中添加日志，记录关联情况

### 预期效果

- 清晰记录 blocks 信息的生成、传递和使用全过程
- 便于调试和监控，快速定位 blocks 丢失的原因
- 不修改核心逻辑，只添加日志，风险可控
- 提供完整的 blocks 信息流转日志链
