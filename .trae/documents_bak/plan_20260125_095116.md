# 实现计划：只生成包含翻译内容的PDF页面

## 1. 问题分析

当前PDF生成器的实现会处理原始PDF的每一页，包括没有翻译内容的页面，这导致生成的PDF包含所有原始页面。根据用户需求，我们需要修改PDF生成器，使其只生成包含翻译内容的页面。

## 2. 实现方案

修改`modules/pdf_generator.py`文件中的`generate_pdf`方法，具体实现如下：

1. **收集有翻译内容的页码**：
   - 遍历`translated_content['blocks']`，收集所有包含翻译内容的页码
   - 同时考虑表格内容，收集包含表格翻译的页码
   - 将这些页码合并为一个集合，确保不重复

2. **只处理有翻译内容的页面**：
   - 遍历原始PDF时，只处理属于上述集合的页码
   - 跳过没有翻译内容的页面
   - 确保新生成的PDF只包含有翻译内容的页面

## 3. 代码修改要点

1. 在`generate_pdf`方法中，添加收集有翻译内容页码的逻辑
2. 修改页面处理循环，只处理指定页码
3. 确保页面编号正确对应

## 4. 测试

- 运行现有测试，确保修改不会破坏现有功能
- 测试不同页码范围的翻译，验证输出PDF只包含指定页码
