# 优化翻译文本拆分函数

## 问题分析
当前拆分算法存在句子太长无法显示的问题，主要是因为：
1. 基于权重的分块逻辑导致块长不均匀
2. 复杂的调整逻辑可能产生不可预测的结果
3. 英文单词拆分和标点处理仍需改进

## 优化方案

### 核心思想
以**合并分块字数向上取整除以分块数**作为基础分块长度，确保每个块的初始长度尽可能均匀，然后进行微调。例如，100个字符分成3块，每个分块34个字符（34*3=102，最后一块会根据实际情况调整）。

### 具体实现步骤

1. **计算基础分块长度**
   - 计算总字符数：`total_chars = len(merged_translation)`
   - 计算基础长度：`base_length = -(-total_chars // num_blocks)`（向上取整）
   - 为每个块分配基础长度

2. **优化拆分逻辑**
   - 从文本开始位置逐块拆分
   - 每块初始长度为base_length
   - 调整拆分位置，确保英文单词完整性和标点正确位置
   - 最后一块处理剩余所有文本

3. **增强英文单词保护**
   - 完善单词边界检测，确保英文单词不被拆分
   - 处理连字符、撇号等特殊情况

4. **严格标点处理**
   - 确保块首不是标点符号
   - 块尾标点保留在当前块
   - 简化标点调整逻辑

5. **简化算法结构**
   - 移除复杂的权重计算
   - 移除最大块长度限制（由基础长度保证均匀性）
   - 简化块间平衡调整

### 预期效果
- 翻译文本拆分更均匀，避免最后一两块过长
- 英文单词保持完整，不被拆分
- 标点符号不出现在块首
- 算法更简单、可靠，易于维护
