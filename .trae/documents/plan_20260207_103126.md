# 修改语义分析提示词，避免Markdown代码块标记

## 问题分析
当前的语义分析提示词虽然要求"仅输出 JSON 格式"，但是LLM仍然返回了包含Markdown代码块标记的结果（```json 和 ```），导致JSON解析失败。

## 解决方案
修改语义分析提示词，使其更加明确地要求大模型只返回纯JSON字符串，不要包含任何Markdown代码块标记或额外格式。

## 实现步骤

### 修改translator.py
- 在 `modules/translator.py` 中的 `_generate_semantic_analysis_prompt` 方法中，修改输出要求部分
- 明确强调以下几点：
  1. 只返回纯JSON字符串，不包含任何其他文本
  2. 不要包含Markdown代码块标记（```json 或 ```）
  3. 确保返回的内容可以直接被JSON解析器解析
  4. 只输出一行JSON，不要有多余的空白行

## 预期效果
- LLM将只返回纯JSON字符串，如 `{"merge":true}`
- 系统能够正确解析LLM返回的结果
- 语义分析失败的错误被修复
- 大模型语义块合并功能能够正常工作
