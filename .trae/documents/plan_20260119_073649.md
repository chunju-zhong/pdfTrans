# 修复翻译失败问题和添加日志记录

## 问题分析
1. **现象**：翻译单页PDF文件时提示"翻译失败: name 'unique_id' is not defined"
2. **根本原因**：
   - 在`translate`函数中定义了`unique_id`变量，用于生成输入文件路径
   - 将`input_filepath`传递给了`process_translation`函数
   - 但在`process_translation`函数中，生成输出文件名时尝试使用`unique_id`变量，该变量未定义
3. **日志问题**：当前代码缺少日志记录，不利于调试和监控

## 修复方案

### 1. 修复NameError问题

**方案**：将`unique_id`和`filename`作为参数传递给`process_translation`函数

**修改步骤**：
- 修改`process_translation`函数签名，添加`unique_id`和`filename`参数
- 修改`translate`函数，传递这两个参数
- 确保`process_translation`函数中使用传递的参数生成输出文件名

### 2. 增加日志记录

**方案**：使用Python标准库`logging`模块，添加日志记录

**修改步骤**：
- 引入logging模块并配置
- 在关键操作点添加日志记录
- 记录错误信息到日志文件
- 配置日志格式和级别

## 实施计划

### 步骤1：修复process_translation函数

1. **修改函数签名**：
   ```python
   def process_translation(task, input_filepath, source_lang, target_lang, translator_type, unique_id, filename):
   ```

2. **确保输出文件名生成正确**：
   ```python
   output_filename = f"translated_{unique_id}_{filename}"
   ```

### 步骤2：修改translate函数

1. **修改线程调用**：
   ```python
   threading.Thread(target=process_translation, args=(task, input_filepath, source_lang, target_lang, translator_type, unique_id, filename)).start()
   ```

### 步骤3：添加日志配置

1. **在app.py顶部添加日志配置**：
   ```python
   import logging
   
   # 配置日志
   logging.basicConfig(
       level=logging.INFO,
       format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
       handlers=[
           logging.FileHandler('app.log'),
           logging.StreamHandler()
       ]
   )
   
   logger = logging.getLogger(__name__)
   ```

2. **在关键操作点添加日志**：
   - 翻译任务开始时
   - 进度更新时
   - 错误发生时
   - 任务完成时

### 步骤4：测试修复

1. **运行测试用例**：确保所有测试通过
2. **手动测试**：使用提供的单页PDF文件测试翻译功能
3. **检查日志**：确认日志记录正常

## 预期效果

1. 修复后，翻译单页PDF文件不再出现"name 'unique_id' is not defined"错误
2. 系统运行时生成日志文件，方便调试和监控
3. 日志包含关键操作和错误信息，便于定位问题
4. 提高系统的可维护性和可靠性
