# 批量语义分析提示词优化方案

## 问题分析
当前批量语义分析提示词存在以下问题：
1. **缺少明确数量要求**：没有明确强调返回结果数量必须与输入文本对数量一致
2. **输出长度要求不具体**：没有明确指定输出数组的长度要求
3. **示例不够详细**：没有提供与输入数量对应的示例，说明数组长度必须与输入数量对应

## 优化方案

### 1. 明确强调返回结果数量要求
- 在输出要求部分明确添加："返回的 `merge` 数组长度必须与输入文本对数量完全一致"
- 强调："每对输入文本必须对应一个分析结果，不得遗漏或重复"

### 2. 添加具体的输出长度要求
- 在输出格式要求中添加："`merge` 数组的长度必须等于输入文本对的数量"
- 明确说明："如果输入 n 对文本，返回的 `merge` 数组必须包含 n 个布尔值"

### 3. 提供更详细的示例
- 添加具体的输入输出对应示例，例如：
  - 输入 3 对文本时，输出应为：`{"merge": [true, false, true]}`
  - 输入 5 对文本时，输出应为：`{"merge": [true, false, true, false, true]}`
- 说明示例中的数组长度与输入数量的对应关系

### 4. 增强错误提示
- 添加："如果返回的结果数量与输入数量不一致，将被视为无效输出"
- 强调："请确保返回的 JSON 格式正确，且 `merge` 数组长度与输入文本对数量一致"

## 实施步骤
1. 修改 `_generate_batch_semantic_analysis_prompt` 方法
2. 更新输出要求部分，添加明确的数量要求
3. 添加具体的输出长度要求
4. 提供更详细的示例，说明数组长度与输入数量的对应关系
5. 增强错误提示，确保 LLM 理解输出要求的严格性
6. 测试验证优化效果

## 预期效果
- LLM 能够正确理解并遵守返回结果数量与输入数量一致的要求
- 批量分析结果数量不一致的问题得到解决
- 系统稳定性和可靠性提高
