## 实现前端文档类型和术语表输入功能计划

### 1. 目标
在前端网页中增加文档类型和术语表输入功能，文档类型默认为"技术文档"，术语表包含默认术语列表，并编写相应的测试用例。

### 2. 实现步骤

#### 2.1 修改前端模板 (`templates/index.html`)
- 在翻译表单中添加文档类型输入字段
- 添加术语表文本区域输入字段
- 设置默认值

#### 2.2 修改后端代码 (`app.py`)
- 在`translate`路由中获取文档类型和术语表参数
- 修改`process_translation`函数签名，添加文档类型和术语表参数
- 在调用翻译器的`translate_with_context`方法时，传递这些参数

#### 2.3 编写测试用例 (`tests/test_app.py`)
- 添加测试用例，验证前端表单提交和后端处理流程
- 测试文档类型和术语表参数的传递
- 测试翻译器方法调用时参数的正确性

#### 2.4 测试验证
- 运行所有测试用例，确保功能正常
- 验证前端表单提交和后端处理流程

### 3. 详细实现

#### 3.1 前端模板修改

```html
<!-- 在源语言和目标语言选择框后添加 -->
<div class="form-group">
    <label for="doc_type">文档类型</label>
    <input type="text" id="doc_type" name="doc_type" value="技术文档" placeholder="请输入文档类型">
</div>

<div class="form-group">
    <label for="glossary">术语表（格式：术语: 翻译，每行一个）</label>
    <textarea id="glossary" name="glossary" rows="6" placeholder="请输入术语表">AI agents: AI智能体
Large Language Model (LLM): 大语言模型（LLM）
paradigm: 范式
passive, discrete tasks: 被动式、离散型任务
prompt: 提示词
autonomous problem-solving: 自主问题解决</textarea>
</div>
```

#### 3.2 后端代码修改

1. 在`translate`路由中添加参数获取：
```python
doc_type = request.form.get('doc_type', '技术文档')
glossary = request.form.get('glossary', '')
```

2. 修改`process_translation`函数签名：
```python
def process_translation(task, input_filepath, source_lang, target_lang, translator_type, unique_id, filename, doc_type="技术文档", glossary=""):
```

3. 在调用翻译器方法时传递参数：
```python
translated_block_text = translator.translate_with_context(
    # ... 现有参数 ...
    doc_type=doc_type,
    glossary=glossary
)
```
