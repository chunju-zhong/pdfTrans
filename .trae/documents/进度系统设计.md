# 进度系统设计说明

## 1. 当前实现分析

### 1.1 现有代码结构

当前进度系统在以下文件中实现：

| 文件 | 作用 |
|------|------|
| `models/task.py` | Task类，管理progress属性和update_progress()方法 |
| `services/translation_service.py` | 翻译流程中的进度更新调用（27处） |
| `app.py` | Flask路由中的进度查询和更新 |
| `static/js/main.js` | 前端进度展示 |

### 1.2 当前问题

1. **硬编码数值过多**：代码中直接使用 `update_progress(30, ...)`, `update_progress(45, ...)` 等硬编码值
2. **进度阶段不明确**：无法清晰看出有哪些进度阶段，每个阶段的范围是什么
3. **维护困难**：调整某个阶段的进度范围需要修改多处代码
4. **语义不清晰**：进度值30表示什么？不直观

### 1.3 当前进度值分布

```
0-10:   初始化阶段 (0, 5, 10)
20-30:  提取阶段 (20, 30)
40-50:  翻译准备/开始阶段 (40, 45, 50)
75-80:  表格翻译阶段 (75, 80)
80-100: 输出生成阶段 (80, 90, 95, 100)
```

## 2. 设计方案

### 2.1 核心概念

引入**进度阶段(Phase)**的概念：

- **Phase（阶段）**：一个独立的工作步骤，如"初始化"、"提取文本"、"翻译"、"生成输出"等
- **PhaseRange（阶段范围）**：每个阶段在整体进度中的百分比范围
- **PhaseProgress（阶段进度）**：某阶段内部的相对进度（0-100%）

### 2.2 数据结构设计

```python
# 进度阶段配置
PHASE_CONFIG = {
    'init': {
        'name': '初始化',
        'start': 0,
        'end': 10,
    },
    'extraction': {
        'name': '文本提取',
        'start': 10,
        'end': 40,
    },
    'translation': {
        'name': '翻译',
        'start': 40,
        'end': 80,
    },
    'table_translation': {
        'name': '表格翻译',
        'start': 80,
        'end': 90,
    },
    'generation': {
        'name': '生成输出',
        'start': 90,
        'end': 100,
    }
}
```

### 2.3 Task类扩展

在 `models/task.py` 中添加：

```python
class Task:
    # 现有属性...
    
    # 新增：当前阶段
    current_phase: str = 'init'
    
    # 新增：阶段配置（可配置）
    phase_config = PHASE_CONFIG
    
    # 方法：更新阶段进度
    def update_phase_progress(self, phase: str, phase_percent: int, message: str = None):
        """根据阶段和阶段内百分比更新整体进度
        
        Args:
            phase: 阶段名称（如'translation'）
            phase_percent: 该阶段内的进度（0-100）
            message: 可选的状态消息
        """
        phase_range = self.phase_config.get(phase)
        if not phase_range:
            return False
            
        # 计算整体进度
        start = phase_range['start']
        end = phase_range['end']
        overall_progress = start + (end - start) * phase_percent // 100
        
        return self.update_progress(overall_progress, message)
```

### 2.4 使用示例

**Before（当前）:**
```python
task.update_progress(30, '正在创建翻译器...')
task.update_progress(45, '正在进行语义合并...')
task.update_progress(50, '语义块合并完成，开始翻译...')
```

**After（设计后）:**
```python
# 方式1：使用阶段更新
task.update_phase_progress('init', 50, '正在创建翻译器...')
task.update_phase_progress('translation', 12, '正在进行语义合并...')
task.update_phase_progress('translation', 25, '语义块合并完成，开始翻译...')

# 方式2：更语义化的方法
task.set_phase('translation')  # 设置当前阶段
task.update_phase_percent(12, '正在进行语义合并...')
```

## 3. 实现步骤

### 3.1 阶段划分

| 阶段ID | 阶段名称 | 进度范围 | 说明 |
|--------|----------|----------|------|
| init | 初始化 | 0-10% | 任务创建、文件检查、翻译器创建 |
| extraction | 文本提取 | 10-40% | PDF文本提取、语义块合并 |
| translation | 翻译 | 40-80% | 文本翻译（主要耗时阶段） |
| table_translation | 表格翻译 | 80-90% | 表格单元格翻译 |
| generation | 生成输出 | 90-100% | PDF/DOCX/MD生成、文件清理 |

### 3.2 实现清单

1. **创建进度阶段配置模块** `models/phase_config.py`
   - 定义 PHASE_CONFIG 常量
   - 提供阶段相关的工具函数

2. **扩展Task类** `models/task.py`
   - 添加 current_phase 属性
   - 添加 phase_config 属性
   - 添加 update_phase_progress() 方法
   - 添加 set_phase() 方法
   - 添加 update_phase_percent() 方法

3. **更新translation_service.py**
   - 将所有硬编码的进度值替换为阶段更新调用

4. **更新app.py**（如需要）
   - 术语提取等功能的进度更新

### 3.3 向后兼容性

- 保留原有的 `update_progress(progress, message)` 方法
- 原有调用方式不受影响
- 新增语义化的阶段更新方法

## 4. 优势

1. **可配置**：调整各阶段进度范围只需修改配置，无需改代码
2. **可扩展**：新增阶段只需在配置中添加一项
3. **语义清晰**：代码表达意图更明确
4. **易于维护**：进度逻辑集中管理
5. **前后端解耦**：前端可以基于阶段显示更友好的进度信息
