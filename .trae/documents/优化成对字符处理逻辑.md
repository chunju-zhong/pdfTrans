# 优化成对字符处理逻辑

## 目标
确保左引号、括号、书名号等成对出现的字符不出现在句尾，提高文本拆分的可读性和正确性。

## 实现步骤

1. **添加辅助函数**：创建一个新的辅助函数 `is_left_pair_character` 来识别不应该出现在句尾的左成对字符

2. **优化 `adjust_split_position` 函数**：
   - 在调整拆分位置时，检查拆分后的块末尾是否为左成对字符
   - 如果是，向前调整拆分位置，直到找到合适的边界
   - 确保调整范围不超过最大调整限制

3. **更新相关逻辑**：
   - 确保在处理块首标点时，正确处理成对字符
   - 保持与现有逻辑的兼容性

## 具体修改

### 1. 添加辅助函数 `is_left_pair_character`
```python
def is_left_pair_character(char):
    """检查字符是否为不应该出现在句尾的左成对字符
    
    Args:
        char (str): 要检查的字符
        
    Returns:
        bool: True如果是左成对字符，否则False
    """
    left_pair_chars = '"\'`([{<（【｛《〈「『'
    return char in left_pair_chars
```

### 2. 修改 `adjust_split_position` 函数
在现有逻辑的基础上，添加对左成对字符的检查：

```python
# 6. 确保拆分位置前的字符不是左成对字符（不应出现在句尾）
while split_pos > 1 and is_left_pair_character(text[split_pos-1]):
    split_pos -= 1
    # 确保不超过最大调整范围
    if original_pos - split_pos > 10:
        break
```

### 3. 测试优化效果
运行现有的测试脚本，确保：
- 左成对字符不出现在句尾
- 英文单词完整性保持
- 标点不出现在块首
- 拆分更均匀，充分利用可用分块长度
