# 完善测试用例计划

## 目标
在不修改现有代码的前提下，创建新的测试文件，将临时测试场景整理成正式的 pytest 测试用例，提高测试覆盖率，特别是覆盖调试过程中发现的各种边缘情况。

## 实现步骤

### 1. 创建新的测试文件
- 创建 `test_text_splitting.py` 文件，专门测试文本拆分功能
- 不修改任何现有代码
- 遵循 pytest 框架规范

### 2. 整理临时测试场景
- 将 `debug_split.py`、`debug_split_optimized.py`、`debug_overflow.py` 和 `debug_pair_chars.py` 中的测试场景整理成正式的 pytest 测试用例
- 确保测试用例结构清晰，易于维护
- 覆盖所有重要的测试场景

### 3. 具体测试用例设计

#### 3.1 测试 `adjust_split_position` 函数
- 测试基本功能：确保返回值在有效范围内
- 测试英文单词不被拆分
- 测试左成对字符不出现在句尾
- 测试标点不出现在块首
- 测试最大调整范围限制

#### 3.2 测试 `split_translated_result` 函数
- 测试合并块包含多个句子的场景
- 测试合并块只有一个句子的场景
- 测试原始块长度差异很大的场景
- 测试英文单词不被拆分到两个块
- 测试标点不作为块首
- 测试中英文混合情况下的拆分
- 测试文本溢出场景下的拆分
- 测试左引号不出现在句尾
- 测试括号不出现在句尾
- 测试书名号不出现在句尾
- 测试多种成对字符混合的场景

#### 3.3 测试 `is_left_pair_character` 函数
- 测试各种左成对字符的识别
- 测试非左成对字符的识别

### 4. 运行测试并记录结果
- 运行所有新创建的测试用例
- 记录无法通过的测试用例，不修改现有代码
- 输出测试报告，说明测试结果

## 预期效果
- 不修改任何现有代码
- 增加测试覆盖率，覆盖更多边缘情况
- 测试用例结构清晰，易于维护和扩展
- 遵循 pytest 框架的规范
- 记录无法通过的测试用例，为后续优化提供参考
