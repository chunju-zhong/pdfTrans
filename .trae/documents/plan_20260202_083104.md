# PDF Extractor 优化方案

## 优化目标

修改 PdfExtractor 类的初始化方法，使其在初始化时输入 PDF 文件路径，提取元数据并保存为实例属性，避免其他方法重复打开文件提取元数据。

## 优化方案

### 1. 修改 `__init__` 方法

**当前实现**：

* 无参数初始化

**修改方案**：

* 添加 `pdf_path` 参数

* 在初始化时调用 `get_metadata` 方法提取元数据

* 将提取的元数据保存为实例属性

* 将 `pdf_path` 保存为实例属性

**具体修改**：

1. 修改 `__init__` 方法签名为 `def __init__(self, pdf_path=None)`
2. 将 `pdf_path` 保存为实例属性 `self.pdf_path`
3. 如果提供了 `pdf_path`，调用 `get_metadata` 方法提取元数据
4. 将提取的元数据保存为实例属性 `self.metadata`
5. 将总页数保存为实例属性 `self.total_pages`

### 2. 优化 `extract_text` 方法

**当前实现**：

* 独立检查文件是否存在

* 直接打开文件获取总页数

**修改方案**：

* 使用实例属性中的元数据，不再重复打开文件

**具体修改**：

1. 使用 `self.pdf_path` 替代方法参数中的 `pdf_path`
2. 使用 `self.metadata` 中的信息，避免重复检查文件存在性
3. 使用 `self.total_pages` 替代直接获取总页数
4. 保留 `fitz.open()` 调用，因为需要获取页面对象进行文本提取

### 3. 优化 `extract_page_text` 方法

**当前实现**：

* 独立检查文件是否存在

* 直接打开文件验证页码范围

**修改方案**：

* 使用实例属性中的元数据，不再重复打开文件

**具体修改**：

1. 使用 `self.pdf_path` 替代方法参数中的 `pdf_path`
2. 使用 `self.metadata` 中的信息，避免重复检查文件存在性
3. 使用 `self.total_pages` 替代直接获取总页数验证页码范围
4. 保留 `fitz.open()` 调用，因为需要获取指定页码的页面对象

### 4. 优化 `extract_tables` 方法

**当前实现**：

* 调用 `extract_tables_impl` 函数，该函数在 `table_processor.py` 中实现

* `extract_tables_impl` 函数中也有独立的文件存在性检查和总页数获取逻辑

**修改方案**：

* 使用实例属性中的元数据，不再重复打开文件

* 考虑修改 `extract_tables_impl` 函数，使其接受总页数参数

**具体修改**：

1. 使用 `self.pdf_path` 替代方法参数中的 `pdf_path`
2. 使用 `self.metadata` 中的信息，避免重复检查文件存在性
3. 使用 `self.total_pages` 替代直接获取总页数
4. 将总页数传递给 `extract_tables_impl` 函数
5. 修改 `extract_tables_impl` 函数，使其接受总页数参数

### 5. 保留 `get_metadata` 方法

**修改方案**：

* 保留 `get_metadata` 方法的功能

* 如果初始化时没有提供 `pdf_path`，其他方法可以在需要时调用 `get_metadata` 方法

## 预期效果

### 性能提升

* **减少文件打开次数**：只在初始化时打开一次文件提取元数据

* **减少文件存在性检查**：只在初始化时检查一次文件存在性

* **减少总页数获取**：只在初始化时获取一次总页数

### 代码质量提升

* **代码一致性**：所有方法使用相同的元数据和总页数

* **代码复用**：充分利用初始化时提取的元数据，避免代码重复

* **异常处理统一**：所有方法使用相同的异常处理逻辑

### 可维护性提升

* **逻辑集中**：元数据提取逻辑集中在初始化方法中

* **易于修改**：如果需要修改元数据提取逻辑，只需要修改初始化方法或 `get_metadata` 方法

* **易于测试**：可以单独测试元数据提取功能

## 风险评估

### 低风险修改

* 修改 `__init__` 方法，添加 `pdf_path` 参数

* 修改其他方法，使用实例属性中的元数据

### 中风险修改

* 修改 `extract_tables_impl` 函数，使其接受总页数参数

### 风险缓解措施

* 在修改前备份原始代码

* 确保修改后的代码通过所有测试用例

* 确保修改后的代码与其他模块的接口保持兼容

* 保持 `get_metadata` 方法的独立功能，以便在需要时可以单独调用
