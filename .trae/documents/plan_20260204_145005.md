# 修复 MergedBlock 对象的边界框访问错误并添加合并边界框计算

## 问题分析
错误发生在 `docx_generator.py` 文件中，当代码尝试使用字典的 `get` 方法访问 `MergedBlock` 对象的 `bbox` 属性时：
- `bbox = block.get('bbox', (0, 0, 0, 0))` 失败，因为 `MergedBlock` 对象没有 `get` 方法
- 更重要的是，`MergedBlock` 对象本身没有 `bbox` 属性

## 解决方案

### 1. 修改 MergedBlock 类，添加 bbox 属性
- 在 `MergedBlock` 类的 `__init__` 方法中添加 `bbox` 属性
- 计算所有原始块的边界框的合并边界：
  - 最小 x0（左上角 x 坐标）
  - 最小 y0（左上角 y 坐标）
  - 最大 x1（右下角 x 坐标）
  - 最大 y1（右下角 y 坐标）
- 更新 `to_dict` 方法，包含 `bbox` 属性

### 2. 更新 docx_generator.py 文件
修改以下两处代码，使用 MergedBlock 对象的 bbox 属性：

#### 2.1 第 127 行附近 - 添加文本块元素
将：
```python
# 尝试获取文本块的边界框
bbox = block.get('bbox', (0, 0, 0, 0))
y_position = bbox[1] if len(bbox) >= 2 else 0
```
改为：
```python
# 获取文本块的边界框
bbox = block.bbox
y_position = bbox[1] if len(bbox) >= 2 else 0
```

#### 2.2 第 207 行附近 - 检查元素是否位于文本块内部
将：
```python
block_bbox = block.get('bbox', (0, 0, 0, 0))
if len(block_bbox) >= 4:
    block_y0, block_y1 = block_bbox[1], block_bbox[3]
```
改为：
```python
block_bbox = block.bbox
if len(block_bbox) >= 4:
    block_y0, block_y1 = block_bbox[1], block_bbox[3]
```

## 预期效果
- 修复 `'MergedBlock' object has no attribute 'get'` 错误
- 为 `MergedBlock` 对象添加 `bbox` 属性，包含所有原始块的合并边界框
- 确保 `docx_generator.py` 文件能够正确处理 `MergedBlock` 对象
- 保持与现有功能的兼容性
