# 修复合并块翻译后拆分不正确的问题

## 问题分析
通过调试发现，`split_translated_result`函数在拆分翻译结果时存在以下问题：
1. 当翻译文本长度较短而原始块数量较多时，只有前面的块能获得内容，后面的块为空
2. 句子边界拆分逻辑会导致一个块占用过多内容，影响后续块的分配
3. 按长度比例计算的分配长度在翻译文本较短时不准确

## 修复方案
改进`split_translated_result`函数的拆分逻辑：

1. **确保每个块至少获得内容**：调整算法，保证每个原始块都能分配到相应的翻译内容
2. **优化句子边界拆分**：限制单次句子边界扩展的长度，避免一个块占用过多内容
3. **平衡分配逻辑**：改进长度比例计算，考虑原始块的实际数量和位置
4. **处理翻译文本长度小于原始块数量的极端情况**

## 修复步骤
1. 修改`app.py`中的`split_translated_result`函数
2. 优化拆分算法，确保每个块至少获得内容
3. 调整句子边界拆分逻辑
4. 运行测试验证修复效果
5. 运行调试脚本验证修复后的拆分行为

## 预期效果
- 每个原始块都能获得对应的翻译内容
- 拆分结果更均匀，符合原始块的长度比例
- 句子边界拆分更加合理，不会导致后续块为空
- 所有测试用例通过
