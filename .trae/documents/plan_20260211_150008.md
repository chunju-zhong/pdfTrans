## Problem Analysis

The current `markdown_generator.py` has a complex chart insertion implementation that differs from the cleaner approach used in `docx_generator.py`. The Markdown generator has unnecessary border calculations and sorting code, and its chart insertion logic is inconsistent with the Word generator.

## Solution

Update the Markdown generator to implement the same chart insertion method based on original block positions as used in the Word generator, ensuring consistency between both generators and simplifying the code structure.

## Implementation Steps

1. **Update the `generate_markdown` method**:
   - Add support for original blocks by extracting them from `translated_content['blocks']`
   - Organize original blocks by page number, similar to the Word generator

2. **Add new helper methods** (matching the Word generator's approach):
   - `_find_chart_position`: Find chart position relative to original text blocks
   - `_find_merged_block`: Find merged block containing chart positions

3. **Update `_process_page_elements` method**:
   - Separate text elements from chart elements
   - Process chart elements to determine their insertion positions
   - Insert charts/tables before/after appropriate merged blocks or at the end

4. **Remove unnecessary code**:
   - Remove complex border calculations
   - Remove unnecessary sorting code
   - Simplify the element processing logic

5. **Ensure consistency**:
   - Use the same chart insertion logic as the Word generator
   - Maintain the same parameter structure and method signatures
   - Ensure both generators handle images and tables the same way

## Expected Outcome

- Markdown and Word generators will have consistent chart insertion logic
- The Markdown generator code will be simpler and easier to maintain
- Both generators will insert charts/tables in the correct positions based on original block locations
- Unnecessary code will be removed, improving performance and readability
