## Problem
The current `merge_semantic_blocks` function merges all text blocks regardless of whether they are body text or not. Non-body text blocks (like headers, footers, and page numbers) should not be merged with other blocks.

## Solution
Modify the `merge_semantic_blocks` function to:
1. Only merge body text blocks with other body text blocks
2. Add non-body text blocks as separate merged blocks
3. Ensure the block mapping remains accurate

1. Update the `merge_semantic_blocks` function in `/Users/chunju/work/pdfTrans/utils/text_processing.py` to:
   - Check if the current block is body text using `curr_text_block.is_body_text`
   - If it's not body text, end the current merge group and add the non-body block as a separate merged block
   - If it's body text, check if it can be merged with the previous body text block
   - Handle edge cases at the beginning and end of the block list

2. Ensure proper handling of:
   - Consecutive non-body text blocks
   - Non-body text blocks at the beginning/end of the list
   - The first block being non-body text
   - Maintaining the correct block mapping

3. Test the changes to ensure:
   - Non-body text blocks are not merged with other blocks
   - Body text blocks are still correctly merged
   - The block mapping remains accurate
