# 实现块按垂直距离排序和序号判断

## 1. 需求分析
- 提取块时按垂直距离排序
- 合并时使用序号即可知道是否是上一块
- 拆分时也可以使用序号判断

## 2. 技术方案

### 2.1 块排序逻辑
- 在收集页面块时，按块的y0值（垂直位置）排序
- 每个页面内的块按垂直方向从上到下排列
- 保持跨页面的顺序不变
- 为每个块添加序号标识，方便后续合并和拆分时使用

### 2.2 合并逻辑优化
- 利用排序后的块序号关系，简化上一块判断
- 保持原有的垂直距离和内容完整性检查
- 通过块的索引位置快速判断是否是上一块

### 2.3 拆分逻辑优化
- 利用块的序号信息，确保拆分结果与原块一一对应
- 保持原有的长度比例拆分算法
- 确保拆分后的结果顺序与原块顺序一致

## 3. 代码修改点

### 3.1 修改 `app.py` 中的块收集逻辑
- 在 `process_translation` 函数中，对每个页面的块按y0值排序
- 排序后再添加到 `all_blocks` 列表中
- 为每个块添加序号标识

### 3.2 优化 `merge_semantic_blocks` 函数
- 利用块的索引位置判断是否是上一块
- 简化垂直距离计算逻辑
- 保持原有的合并条件

### 3.3 优化 `split_translated_result` 函数
- 利用块的序号信息，确保拆分结果与原块一一对应
- 保持原有的长度比例拆分算法
- 确保拆分后的结果顺序与原块顺序一致

## 4. 预期效果
- 块按垂直方向从上到下排序
- 合并时通过块的序号即可判断是否是上一块
- 拆分时通过块的序号确保结果与原块一一对应
- 保持原有的合并和拆分逻辑效果
- 提高代码的可读性和效率
