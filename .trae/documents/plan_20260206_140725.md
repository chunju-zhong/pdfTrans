# 为 analyze_semantic_relationship 方法添加语言类型参数

## 问题分析
当前 `analyze_semantic_relationship` 方法没有考虑语言类型，这可能会影响语义分析的准确性。根据用户要求，需要为该方法添加语言类型参数，以便在语义分析时考虑语言特性。

## 解决方案
1. **修改 Translator 基类**
   - 在 `modules/translator.py` 中修改 `analyze_semantic_relationship` 方法，添加 `source_lang` 参数
   - 修改 `_generate_semantic_analysis_prompt` 方法，添加 `source_lang` 参数，并在提示词中使用

2. **修改 AipingTranslator 子类**
   - 在 `modules/aiping_translator.py` 中修改 `analyze_semantic_relationship` 方法，添加 `source_lang` 参数

3. **修改 SiliconFlowTranslator 子类**
   - 在 `modules/silicon_flow_translator.py` 中修改 `analyze_semantic_relationship` 方法，添加 `source_lang` 参数

4. **修改 merge_semantic_blocks_with_llm 函数**
   - 在 `utils/text_processing.py` 中修改 `merge_semantic_blocks_with_llm` 函数，添加 `source_lang` 参数
   - 在调用 `analyze_semantic_relationship` 时传递该参数

5. **修改 translation_service.py 中的调用**
   - 在 `services/translation_service.py` 中修改 `merge_semantic_blocks_with_llm` 的调用，传递 `source_lang` 参数

6. **测试验证**
   - 运行测试确保修改后的代码正常工作

## 实现细节
- 在基类的 `_generate_semantic_analysis_prompt` 方法中，将语言类型信息添加到提示词中，例如："这两个文本块是{语言名称}，请根据{语言名称}的语法和语义规则进行分析"
- 保持原有的分析逻辑和返回值不变
- 确保所有调用链都正确传递语言类型参数
