# 优化PDF翻译流程：源语言与目标语言相同时直接拷贝原始页

## 问题分析
当前PDF翻译工具在源语言和目标语言相同时，仍然执行完整的翻译流程：
1. 提取PDF文本
2. 语义块合并
3. 翻译（返回原文）
4. 翻译结果拆分
5. 生成新PDF（克隆原始页并绘制原文）

这造成了不必要的资源消耗和时间浪费。

## 优化方案
在`translation_service.py`的`process_translation`方法中添加源语言和目标语言的比较，当两者相同时：
1. 跳过所有文本处理和翻译步骤
2. 直接将原始PDF页面拷贝到输出文件
3. 更新任务状态和进度
4. 清理临时文件

## 实现步骤
1. 在`process_translation`方法开头添加源语言和目标语言的比较
2. 如果相同，直接创建输出文件路径
3. 使用文件拷贝或PyMuPDF的页面拷贝功能将原始PDF复制到输出文件
4. 更新任务状态为完成
5. 清理临时文件
6. 返回结果

## 预期效果
- 当源语言和目标语言相同时，翻译速度显著提升
- 减少不必要的资源消耗
- 提高用户体验
- 保持功能完整性
