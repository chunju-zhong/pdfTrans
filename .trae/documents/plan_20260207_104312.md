# 优化Markdown生成器的提示词，避免返回代码块标记

## 问题分析
当前的Markdown生成器使用大模型将文本转换为Markdown格式，但提示词没有明确要求LLM不要返回 ```markdown ``` 代码块标记，可能导致生成的Markdown文档包含不必要的代码块包装。

## 解决方案
修改 `_load_layout_prompt` 方法中的系统提示词，明确要求LLM只返回纯Markdown文本，不要包含代码块标记。

## 实现步骤

### 修改markdown_generator.py
- 在 `modules/markdown_generator.py` 中的 `_load_layout_prompt` 方法中，在最终输出规范部分添加明确的要求：
  1. 只返回纯Markdown文本，不包含任何代码块标记
  2. 不要使用 ```markdown 或 ``` 包装输出
  3. 确保返回的内容可以直接保存为Markdown文件

## 预期效果
- LLM将只返回纯Markdown文本，不包含代码块标记
- 生成的Markdown文档格式正确，没有不必要的代码块包装
- 系统能够正确处理LLM返回的结果，无需额外的标记移除处理
