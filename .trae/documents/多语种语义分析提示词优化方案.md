# 多语种语义分析提示词优化方案

## 问题分析
当前批量语义分析提示词存在以下问题：
1. **列表项误识别**："of components："被误识别为列表项
2. **语法延续性检测不足**："a variety" + "of"这样的常见短语延续未被充分考虑
3. **语种适应性差**：未针对不同语种的语法特点进行优化

## 优化方案

### 1. 修改列表项识别规则
- **明确列表项定义**：只有带编号或符号的文本块才是列表项
- **区分列表开始标记**："of components："这样的短语是列表开始标记，不是列表项
- **添加列表项特征**：列表项通常以数字、字母、符号开头，且表达独立项目

### 2. 增强语法延续性检测
- **添加常见短语延续规则**：
  - 英语："a variety" + "of", "such" + "as", "for example" + ":"
  - 中文："一系列" + "的", "例如" + "：", "包括" + "以下"
  - 其他语种：根据各语种特点添加常见短语延续
- **添加语法结构检测**：
  - 前一个块以介词、冠词结尾
  - 后一个块以名词、动词开头
  - 整体构成完整语法结构

### 3. 调整判断逻辑优先级
1. **标题判断**（最高优先级）
2. **语法延续性判断**（新增，第二优先级）
3. **列表项判断**（第三优先级）
4. **语义单元判断**（最低优先级）

### 4. 多语种优化
- **英语**：重点优化短语延续和标点使用
- **中文**：重点优化语序和标点使用
- **其他语种**：根据各语种语法特点进行针对性优化

## 实施步骤
1. 修改`_generate_batch_semantic_analysis_prompt`方法
2. 更新列表项识别规则
3. 添加语法延续性检测规则
4. 调整判断逻辑优先级
5. 针对多语种进行优化
6. 测试验证优化效果

## 预期效果
- 正确识别并合并"a variety" + "of components："这样的短语延续
- 准确区分列表项和列表开始标记
- 提高多语种语义分析的准确性
- 减少误判率，提高合并质量
