# 去掉降级方案，直接返回错误提示用户

## 问题分析
当前系统在大模型调用失败时会使用原始文本作为降级方案，用户要求去掉这些降级方案，直接返回错误提示用户。

## 解决方案
修改系统以在大模型调用失败时，直接抛出异常并返回错误信息，而不是使用原始文本作为降级方案。

## 实现步骤

### 1. 修改Markdown生成器
- 在 `modules/markdown_generator.py` 中的 `_format_with_layout_model` 方法中，当布局模型请求失败时，不再返回原始文本
- 改为抛出异常，让上层调用处理错误
- 移除 "使用原始文本作为降级方案继续生成Markdown文档" 的日志信息

### 2. 修改翻译服务
- 在 `services/translation_service.py` 中的 `generate_output_files` 方法中，捕获Markdown生成器可能抛出的异常
- 当捕获到异常时，调用 `task.set_error()` 设置错误状态
- 确保错误信息能够传递给前端

### 3. 修改前端错误处理
- 确保前端能够正确显示翻译过程中的错误信息
- 在 `static/js/main.js` 中的 `updateProgress` 函数中，当任务状态为ERROR时，显示错误消息

## 预期效果
- 当系统正常完成翻译时，前端显示"翻译完成！"
- 当大模型调用失败时，系统不再使用降级方案，而是直接返回错误信息
- 前端能够清晰地显示错误原因，用户可以了解失败的具体原因
- 系统行为更加明确，不会在失败时静默使用降级方案
