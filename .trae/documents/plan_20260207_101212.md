# 修复语义分析失败的错误

## 问题分析
在 `modules/translator.py` 文件的 `_generate_semantic_analysis_prompt` 方法中，第 197 行存在错误的 Python 字符串格式化语法：

```python
{"merge": true/false}
```

这里的 `{}` 被 Python 解释为格式说明符，但 `"merge": true/false` 不是有效的格式说明符，导致出现 `Invalid format specifier ' true/false' for object of type 'str'` 错误。

## 解决方案
修复字符串格式化语法，正确生成语义分析提示词。

## 实现步骤

### 修改翻译器基类
- 在 `modules/translator.py` 中的 `_generate_semantic_analysis_prompt` 方法中，修复 JSON 格式的字符串格式化问题
- 将 `{"merge": true/false}` 修改为 `{{"merge": true/false}}`，使用双大括号转义
- 确保生成的提示词格式正确，能够被翻译器正确处理

## 预期效果
- 语义分析失败的错误被修复
- 系统能够正确生成语义分析提示词
- 大模型能够正确分析文本块之间的语义关系
- 翻译服务能够正常使用 LLM 进行语义块合并
