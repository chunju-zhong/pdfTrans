# 修复方案：解决PDF生成时document closed错误

## 1. 问题分析

从日志中可以看到，错误发生在`pdf_generator.py`文件的第161行：
```
logger.info(f"PDF生成完成，输出文件: {output_pdf_path}, 总页数: {len(new_doc)}")
```

错误信息：`ValueError: document closed`

在PyMuPDF中，当调用`doc.save()`方法时，文档会在保存后自动关闭，这就是为什么在保存后尝试访问文档的`len()`属性时会抛出错误。

## 2. 修复方案

修改`modules/pdf_generator.py`文件中的`generate_pdf`方法，在文档保存之前获取总页数，然后在日志中使用这个预存的总页数，避免在文档关闭后访问它。

## 3. 代码修改要点

1. 在保存文档之前获取总页数
2. 保存文档
3. 使用预存的总页数记录日志，避免在文档关闭后访问它

## 4. 测试

运行现有的测试用例，验证修复后的功能是否正常工作
