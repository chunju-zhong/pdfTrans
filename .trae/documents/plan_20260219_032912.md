# 批量语义分析提示词详细修改方案

## 问题分析

当前提示词存在以下问题：

1. **列表项规则不完整**：没有规定"如果块1是列表项，块2是非列表项"的情况
2. **规则优先级不明确**：语法延续性规则优先级可能过高
3. **列表项处理逻辑不一致**：对于列表项的后续内容处理规则不明确
4. **语义理解偏差**：没有明确说明列表项可以跨多个文本块

## 修改方案

### 1. 完善列表项处理规则

#### 修改列表项识别规则

* **明确列表项定义**：列表项是带编号或符号开头的文本块，表达独立项目

* **添加列表项延续规则**：列表项可以跨多个文本块，后续的延续内容应与列表项合并

* **区分列表项开头和延续**：

  * 列表项开头：以编号或符号开头

  * 列表项延续：非编号或符号开头，是列表项开头的延续

#### 新增列表项延续识别规则

* **特征**：非编号或符号开头，紧跟在列表项开头之后

* **语法**：通常是列表项开头的语法延续

* **示例**：

  * 列表项开头："• Context to guide reasoning defines the agent’s fundamental reasoning patterns and"

  * 列表项延续："available actions, dictating its behavior:"

### 2. 调整规则优先级

#### 重新定义判断逻辑优先级

1. **标题判断**（最高优先级）
2. **列表项判断**（第二优先级）

   * 如果块2是列表项（新的列表项），返回: false

   * 如果块1是列表项，块2是非列表项且是其延续，返回: true
3. **语法延续性判断**（第三优先级）
4. **逻辑句子判断**（最低优先级）

#### 修改具体规则

* **规则4**：修改为"如果块2是列表项（带编号或符号的独立项目），返回: false"

* **新增规则**："如果块1是列表项，块2是非列表项且是其延续，返回: true"

* **规则5**：保持不变，但优先级调整为低于列表项判断

### 3. 增强语义理解

#### 添加列表项延续示例

* **示例1**：

  * 块1: "• Context to guide reasoning defines the agent’s fundamental reasoning patterns and"

  * 块2: "available actions, dictating its behavior:"

  * 分析结果: true（应该合并）

* **示例2**：

  * 块1: "available actions, dictating its behavior:"

  * 块2: "• System Instructions: High-level directives defining the agent's persona, capabilities,"

  * 分析结果: false（不应该合并）

#### 明确语义单元划分

* **列表项语义单元**：列表项开头及其延续内容构成一个完整的语义单元

* **新列表项**：以编号或符号开头的文本块开始一个新的语义单元

### 4. 优化规则表述

#### 重新组织判断逻辑部分

* **更清晰的规则结构**：按照优先级顺序排列规则

* **更精确的规则描述**：明确每个规则的适用条件和返回值

* **更详细的示例**：提供涵盖各种情况的示例

#### 增强规则的可理解性

* **使用明确的术语**：区分"列表项开头"、"列表项延续"、"新列表项"

* **使用一致的表述**：确保规则描述与示例一致

* **使用简洁的语言**：避免歧义性表述

## 实施步骤

1. **修改判断逻辑部分**：重新组织规则优先级，完善列表项处理规则
2. **添加列表项延续识别规则**：明确列表项可以跨多个文本块
3. **更新示例**：添加列表项延续的具体示例
4. **优化规则表述**：提高规则的清晰度和可理解性
5. **测试验证**：使用实际日志中的文本对6和文本对7进行测试

* **文本对7**：正确判断为不应该合并（块2是新的列表项）

* **其他文本对**：保持正确的判断结果

* **整体效果**：提高列表项相关文本对的判断准确性
