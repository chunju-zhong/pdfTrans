# 文本块章节归属准确性修复 - 产品需求文档

## Overview

* **Summary**: 修复PDF翻译工具中文本块章节归属不准确的问题，特别是对于跨章节页的处理，确保文本块能根据其垂直位置正确归属到相应章节。

* **Purpose**: 解决当前基于整页的章节归属逻辑导致的错误归属问题，提高章节划分的准确性。

* **Target Users**: PDF翻译工具的用户，特别是需要按章节生成翻译结果的用户。

## Goals

* 提高文本块章节归属的准确性，特别是对于跨章节页

* 实现基于文本块垂直位置的章节归属逻辑

* 保持与现有代码的兼容性

* 提供清晰的日志记录，便于调试和验证

## Non-Goals (Out of Scope)

* 不修改现有的章节识别逻辑

* 不改变章节树的构建方式

* 不影响其他功能模块的正常运行

## Background & Context

当前的章节归属逻辑在 `ChapterIdentifier._build_chapter_mapping()` 方法中实现，它：

1. 按页码排序所有章节
2. 为每个章节分配一个页码范围：从当前章节的起始页码到下一个章节的起始页码减1
3. 最后一个章节的结束页码设为一个很大的值（999999）
4. 为范围内的每个页码分配章节

这种方法是基于整页的章节归属，而不是基于文本块的垂直位置和PDF书签的位置数据，导致了以下问题：

* 对于跨章节页，比如第11页的后段属于4小节，第12页属于4.1小节，当前逻辑会将第11页的所有文本块都归为4小节

* 可能将页面错误地归为相邻章节，比如4.4小节应该只有17-18页，却把19页也归到了4.4

PDF书签通常包含详细的位置信息（页码和页面坐标），这些信息可以用来精确定位章节标题在页面上的位置，从而更准确地判断文本块的章节归属。

## Functional Requirements

* **FR-1**: 只使用PDF书签页码信息和页面位置数据（如x, y坐标）精确定位章节归属

* **FR-2**: 实现基于文本块垂直位置与书签位置数据对比的章节归属逻辑

* **FR-3**: 准确处理跨章节页的文本块归属

* **FR-4**: 保持与现有代码的兼容性

* **FR-5**: 提供详细的日志记录，便于调试和验证

## Non-Functional Requirements

* **NFR-1**: 性能影响最小化，不显著增加提取时间

* **NFR-2**: 代码可读性和可维护性良好

* **NFR-3**: 向后兼容，不破坏现有功能

## Constraints

* **Technical**: 基于现有的PyMuPDF库和章节识别逻辑

* **Dependencies**: 依赖现有的ChapterIdentifier类和TextBlock类

## Assumptions

* PDF文档包含有效的书签信息（包括页页与起始位置），用于章节识别

* 文本块的垂直位置（y坐标）可以用来判断其章节归属

## Acceptance Criteria

### AC-1: 跨章节页文本块归属准确性

* **Given**: PDF文档包含跨章节页，比如第11页的后段属于4小节，第12页属于4.1小节

* **When**: 提取文本块并关联章节信息

* **Then**: 第11页的文本块应根据其垂直位置正确归属到4小节或4.1小节

* **Verification**: `programmatic`

### AC-2: 错误页码归属修复

* **Given**: PDF文档中4.4小节应该只有17-18页

* **When**: 提取19页的文本块并关联章节信息

* **Then**: 19页的文本块应归属于正确的章节，而不是4.4小节

* **Verification**: `programmatic`

### AC-3: 性能影响最小化

* **Given**: 处理包含多个章节的PDF文档

* **When**: 提取文本块并关联章节信息

* **Then**: 处理时间不应显著增加（不超过10%）

* **Verification**: `programmatic`

### AC-4: 向后兼容性

* **Given**: 处理不包含跨章节页的PDF文档

* **When**: 提取文本块并关联章节信息

* **Then**: 章节归属结果应与之前一致

* **Verification**: `programmatic`

## Open Questions

* [x] 如何准确识别跨章节页上的章节标题位置？

  * **解决方案**：使用PDF书签页码信息和页面位置数据（如x, y坐标）精确定位章节归属

    1. **获取详细书签信息**：

       * 使用PyMuPDF的`doc.get_toc(simple=False)`方法或直接访问PDF大纲对象，获取包含位置数据的详细书签信息

       * 书签信息应包含章节标题、级别、页码和页面位置坐标（x, y）
    2. **精确定位章节标题**：

       * 利用书签的位置坐标，精确定位章节标题在页面上的位置

       * 对于跨章节页，根据书签位置坐标确定章节边界
    3. **文本块归属**：

       * 根据文本块的垂直位置（y坐标）与章节标题位置的比较，确定文本块的章节归属

       * 文本块归属于其上方最近的章节标题

* [x] 如何处理页面上多个章节的情况？

  * **解决方案**：

    1. 按垂直位置排序所有章节标题
    2. 为每个文本块根据其垂直位置归属到相应的章节
    3. 文本块归属于其上方最近的章节标题

* [x] 如何平衡准确性和性能之间的关系？

  * **解决方案**：

    1. 缓存章节信息，减少重复计算
    2. 只对可能包含跨章节的页面进行详细分析
    3. 使用高效的算法进行文本块归属计算
